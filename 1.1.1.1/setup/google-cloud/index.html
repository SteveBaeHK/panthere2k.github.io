Set up 1.1.1.1 - Google Cloud

Google Cloud supports configuring outbound server policy Open external link within Cloud DNS. Policies are applied per Virtual Private Cloud (VPC) network, and will affect all resources within that VPC network, including any existing virtual machines.

If you are using Cloudflare Zero Trust, you can choose assigned locations to apply custom DNS policies via Gateway.

To configure 1.1.1.1 for your Google Cloud VPC network(s):

  1. Open the Google Cloud Console Open external link .
  2. Navigate to Network Services > Cloud DNS and select DNS Server Policies Open external link .
  3. Select Create Policy.
  4. Provide a name for your Policy (such as cloudflare-1-1-1-1) and select associated VPC network or networks.
  5. Under Alternate DNS servers, select Add Item and type:
    1.1.1.1
    1.0.0.1
  6. Select Create.

DNS requests within the configured VPC network(s) will now use 1.1.1.1.
